# Core Documentation

Evergreen human-readable docs for operating and maintaining this project.

## Files
- `engineering_guide.md`
  - coding, testing, and workflow standards.
- `runbook.md`
  - deploy, promote, rollback, and validation flow.
- `command_reference.md`
  - quick command lookup.
- `security.md`
  - baseline security controls, secrets, CSP, and logging safety.
- `architecture.md`
  - current production architecture and major technical decisions.
- `convention_ops.md`
  - convention-specific runtime and image operations.
- `ownership_and_slos.md`
  - contract ownership model and service-level targets.

## Lifecycle Rules
- Keep these docs concise and current.
- Add to `docs/active/` for temporary initiatives, then archive when complete.
- Do not add duplicate policy/runbook files under new names.
